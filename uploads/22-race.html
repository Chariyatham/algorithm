เปรียบเทียบ 5 sorting algorithms วิ่งบน array ตัวเดียวกัน — เห็นใครชนะ ใครแพ้ นับ comparisons + swaps ทุกขั้นตอน

รอเริ่ม race

💡 ดูยังไงให้เข้าใจ

  • Random input: ทุก sort วิ่งเหมือนทฤษฎี — Merge/Quick ชนะ Bubble/Insertion ชัดเจน
  • Sorted input: Insertion เร็วที่สุด (O(n) best case), Quick worst case O(n²)!
  • Reversed: Insertion worst (O(n²)), แต่ละ insert เลื่อนทุกตัวก่อนหน้า
  • Few unique: ดูจำนวน swap น้อย แต่ comparison เยอะ
  • Nearly sorted: Insertion ชนะแบบสบาย ๆ — เพราะแต่ละ element ต้องเลื่อนน้อย

Insight: "ดีที่สุด" ขึ้นกับ input — ไม่มี one-size-fits-all นี่คือเหตุผลที่ std::sort() ใช้ introsort (quick + heap + insertion ผสม)
