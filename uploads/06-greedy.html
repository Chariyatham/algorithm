Greedy — เลือกตัวที่ดีที่สุดในตอนนั้นเลย

Greedy = ในแต่ละ step เลือก option ที่ดูดีที่สุด โดยไม่ย้อนแก้ เร็วกว่า DP มาก แต่ ไม่ได้ใช้ได้กับทุกปัญหา — ต้องพิสูจน์ก่อนว่าใช้ได้

Greedy ใช้ได้เมื่อไหร่?

ปัญหาต้องมี 2 คุณสมบัติ:

  1. Greedy choice property — เลือก local optimum แล้วยังไปสู่ global optimum ได้
  2. Optimal substructure — เมื่อตัดสินใจ greedy choice แล้ว ปัญหาที่เหลือยังเป็นปัญหา type เดียวกันในขนาดเล็กลง (เหมือน DP)

กล่าวคือ — เลือกถูกครั้งแรกทุกครั้ง คุณก็จะไม่ผิด

ตัวอย่างที่ Greedy ทำได้: Fractional Knapsack

ของแต่ละชิ้น หั่นเศษได้ — เลือกใส่กระเป๋า W kg ให้ value สูงสุด

Greedy strategy: เรียงตาม value/weight (ความคุ้มค่าต่อกิโล) จากมากไปน้อย แล้วใส่ตามลำดับ

struct Item { double w, v; };

double fractionalKnapsack(vector<Item>& items, double W) {
    sort(items.begin(), items.end(), [](Item a, Item b) {
        return a.v / a.w > b.v / b.w;          // เรียง ratio มาก→น้อย
    });
    double total = 0;
    for (auto& it : items) {
        if (W >= it.w) {                       // ใส่ทั้งชิ้น
            total += it.v; W -= it.w;
        } else {                                // ใส่แค่บางส่วน
            total += it.v * (W / it.w); break;
        }
    }
    return total;
}
// O(n log n) จาก sort

ตัวอย่างที่ 2 — Activity Selection

มี activity n อย่าง แต่ละอย่างมี start, finish — เลือกให้ ไม่ทับกัน ได้มากสุดกี่อย่าง?

Greedy: เรียงตาม finish time น้อย→มาก แล้วเลือกอันที่ start ≥ finish ของอันก่อนหน้า

struct Act { int s, f; };

int activitySelection(vector<Act>& A) {
    sort(A.begin(), A.end(), [](Act a, Act b) { return a.f < b.f; });
    int count = 1, lastFinish = A[0].f;
    for (int i = 1; i < A.size(); i++) {
        if (A[i].s >= lastFinish) {
            count++;
            lastFinish = A[i].f;
        }
    }
    return count;
}
// O(n log n)
รอเริ่ม

ทำไม "เรียงตาม finish" ใช้ได้? เพราะ activity ที่จบเร็วที่สุดจะ เปิดที่ว่าง ให้ activity อื่นมากที่สุด — proof แบบ exchange argument

ตัวอย่างที่ 3 — Coin Change (greedy ใช้ได้บางกรณี)

เหรียญไทย {1, 5, 10, 25} ทอน 30: เริ่ม 25 → เหลือ 5 → 5 = สอง 1 เหรียญ → 25+5

int greedyCoin(vector<int>& coins, int amount) {
    sort(coins.rbegin(), coins.rend());        // มาก→น้อย
    int count = 0;
    for (int c : coins) {
        count += amount / c;
        amount %= c;
    }
    return (amount == 0) ? count : -1;
}
กับดัก! Greedy ใช้กับ coin set มาตรฐาน เท่านั้น (1, 5, 10, 25, ...)
ลอง coins = {1, 3, 4}, amount = 6:
  • Greedy: 4 + 1 + 1 = 3 เหรียญ
  • Optimal: 3 + 3 = 2 เหรียญ ✓
coin set แบบ "non-canonical" ต้องใช้ DP

ตัวอย่างที่ 4 — Huffman Coding

Compress ข้อความ — ให้ตัวอักษรที่ปรากฏบ่อยมีรหัสสั้น ตัวอักษรที่หายากมีรหัสยาว

Greedy: ทุก step เอา 2 node ที่ frequency น้อยสุดมารวมเป็น tree node ใหม่

struct HNode { int freq; char ch; HNode *l, *r; };

struct Cmp { bool operator()(HNode* a, HNode* b) { return a->freq > b->freq; } };

HNode* buildHuffman(vector<pair<char,int>>& freqs) {
    priority_queue<HNode*, vector<HNode*>, Cmp> pq;
    for (auto& [c, f] : freqs)
        pq.push(new HNode{f, c, nullptr, nullptr});

    while (pq.size() > 1) {
        HNode* x = pq.top(); pq.pop();
        HNode* y = pq.top(); pq.pop();
        HNode* parent = new HNode{x->freq + y->freq, '\0', x, y};
        pq.push(parent);
    }
    return pq.top();
}
// O(n log n)

ตัวอย่างที่ 5 — Minimum Spanning Tree (Kruskal)

ใน graph มีน้ำหนัก — หา tree ที่เชื่อมทุก node โดยรวมน้ำหนักน้อยสุด

Kruskal's algorithm: เรียง edge น้อย→มาก แล้วเลือก edge ที่ ไม่สร้าง cycle ใช้ union-find เช็ค

struct Edge { int u, v, w; };

int find(vector<int>& p, int x) {
    while (p[x] != x) { p[x] = p[p[x]]; x = p[x]; }
    return x;
}

int kruskal(int n, vector<Edge>& edges) {
    sort(edges.begin(), edges.end(), [](Edge a, Edge b) { return a.w < b.w; });
    vector<int> parent(n);
    iota(parent.begin(), parent.end(), 0);   // [0,1,2,...,n-1]
    int total = 0, used = 0;
    for (auto& e : edges) {
        int ru = find(parent, e.u), rv = find(parent, e.v);
        if (ru != rv) {                       // ไม่สร้าง cycle
            parent[ru] = rv;
            total += e.w; used++;
            if (used == n - 1) break;
        }
    }
    return total;
}
// O(E log E)

วิธีพิสูจน์ Greedy ถูก (formal)

  1. Exchange argument: สมมติมี optimal solution ที่ไม่ใช้ greedy choice แล้วแสดงว่าเปลี่ยน choice ใน optimal เป็น greedy choice ก็ยัง optimal
  2. Greedy stays ahead: หลัง k step แรก greedy choice ดีกว่าหรือเทียบเท่า choice ใด ๆ ก็ตาม

ลำดับการเลือก algorithm ในชีวิตจริง

  1. ลอง greedy ก่อนเสมอ — คิดง่าย, code สั้น, เร็ว
  2. หา counter-example ขนาดเล็ก — ถ้ามี greedy ใช้ไม่ได้
  3. ถ้า greedy ไม่ work → ลอง DP
  4. ถ้า DP state ใหญ่เกิน → backtracking + pruning

Q1 Job Scheduling: มี job n อย่าง แต่ละอันมี deadline และ profit ทำได้ทีละ 1 job ใน 1 หน่วยเวลา หา total profit สูงสุดโดยทำทันก่อน deadline ของทุก job ที่เลือก
เฉลย — Greedy: เรียงตาม profit มาก→น้อย 
struct Job { int id, deadline, profit; };

int jobScheduling(vector<Job>& jobs) {
    sort(jobs.begin(), jobs.end(), [](Job a, Job b) {
        return a.profit > b.profit;
    });
    int maxD = 0;
    for (auto& j : jobs) maxD = max(maxD, j.deadline);
    vector<int> slot(maxD + 1, -1);
    int total = 0;
    for (auto& j : jobs) {
        // หา slot ว่างที่ใกล้ deadline ที่สุด
        for (int t = j.deadline; t >= 1; t--) {
            if (slot[t] == -1) {
                slot[t] = j.id;
                total += j.profit;
                break;
            }
        }
    }
    return total;
}
Q2 Gas Station: รอบวงกลมมี gas station ที่จุดที่ i มีน้ำมัน gas[i] เดินทางจาก i ไป i+1 ใช้ cost[i] หา start point ที่ทำได้ครบรอบ (ถ้ามี)
เฉลย 
int canCompleteCircuit(vector<int>& gas, vector<int>& cost) {
    int total = 0, tank = 0, start = 0;
    for (int i = 0; i < gas.size(); i++) {
        int diff = gas[i] - cost[i];
        total += diff;
        tank += diff;
        if (tank < 0) {                   // ผ่าน i ไม่ได้
            start = i + 1;                 // ลอง start ใหม่
            tank = 0;
        }
    }
    return (total >= 0) ? start : -1;
}
// O(n)

เหตุผล: ถ้ารวมทั้งวง total < 0 → เป็นไปไม่ได้ ถ้ารวม ≥ 0 → start ที่จุดที่ tank กลับเป็นบวกได้เสมอ

Q3 เหตุใด greedy ไม่ work กับ 0/1 Knapsack แต่ work กับ Fractional?
เฉลย

Fractional: หั่นเศษได้ — ถ้าเหลือพื้นที่ 5 kg เราใส่ 5 kg ของ ratio สูงสุดได้ → ใช้ทุกหยดของ capacity

0/1: เอาทั้งชิ้นหรือไม่เอาเลย — บางทีของ ratio สูงทำให้พื้นที่เหลือ "ใช้ไม่ได้" เลยพลาดโอกาสรวมของ ratio ต่ำกว่า 2 ชิ้นที่จุได้พอดีกับขีดจำกัด

กล่าวคือ Fractional ไม่มี "พื้นที่เหลือเสีย"
