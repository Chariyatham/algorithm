Big O & Time Complexity

"อัลกอริทึมนี้เร็วแค่ไหน?" คำตอบไม่ใช่วินาที — เพราะคอมแต่ละเครื่องเร็วต่างกัน เราวัดเป็น จำนวน operation ที่ใช้ตามขนาด input n ต่างหาก

หลักคิดใน 30 วินาที

สมมติเรามี input ขนาด n เราถามว่า "ถ้า n โตขึ้น เวลาทำงานโตเร็วแค่ไหน?"

  • ถ้าทำงานเสร็จใน เวลาคงที่ เสมอ → O(1)
  • ถ้าวิ่ง loop n รอบ → O(n)
  • ถ้า loop ซ้อน loop → O(n²)
  • ถ้าแบ่งครึ่งทุกครั้ง (เช่น binary search) → O(log n)
  • ถ้าลองทุก subset → O(2ⁿ)

Big O คือ upper bound แบบหยาบ — เราตัดค่าคงที่ทิ้ง เช่น 3n+5 → O(n), 2n²+100n → O(n²)

เห็นภาพ: กราฟเปรียบเทียบ

หาความซับซ้อนจากโค้ดยังไง?

กฎที่ใช้ได้ 95% ของกรณี:

  1. Loop เดี่ยวที่วน n ครั้ง = O(n)
  2. Loop ซ้อน 2 ชั้น แต่ละชั้นวน n = O(n²)
  3. หารครึ่งทุกรอบ (เช่น n /= 2) = O(log n)
  4. ลำดับการทำงาน A แล้ว B = O(A + B) → เอา dominate ที่ใหญ่กว่า
  5. เรียก function recursive 2 ครั้ง ที่ลด n ลง 1 = O(2ⁿ) (เช่น Fibonacci ดิบ ๆ)

ตัวอย่างวิเคราะห์

ตัวอย่าง 1 — O(n)

int sumArray(vector<int>& a) {
    int sum = 0;                  // O(1)
    for (int i = 0; i < a.size(); i++)  // วน n ครั้ง
        sum += a[i];              // O(1) ต่อรอบ
    return sum;
}
// รวม: O(1) + O(n)·O(1) = O(n)

ตัวอย่าง 2 — O(n²)

// หาคู่ที่ผลรวม = target (วิธีดิบ)
for (int i = 0; i < n; i++)        // n รอบ
    for (int j = i+1; j < n; j++)  // ภายในรอบ i: n-i-1 รอบ
        if (a[i] + a[j] == target) ...
// รวม operations: n + (n-1) + ... + 1 = n(n-1)/2 → O(n²)
เคล็ด: loop ซ้อนไม่ได้แปลว่า O(n²) เสมอ! ดูว่า loop ในวิ่งกี่รอบจริง ๆ ถ้า inner loop วิ่ง log n → ทั้งหมดเป็น O(n log n)

ตัวอย่าง 3 — O(log n)

int binarySearch(vector<int>& a, int target) {
    int lo = 0, hi = a.size() - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
// แต่ละรอบช่วงค้นหาลด ½ → จำนวนรอบ = log₂ n

ตัวอย่าง 4 — O(2ⁿ) (Fibonacci ดิบ)

int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);  // เรียกตัวเอง 2 ครั้ง
}
// ขนาด tree ของ recursion ≈ 2ⁿ → ช้ามาก!
// fib(40) ใช้เวลาเป็นวินาที, fib(50) เป็นนาที

แบบฝึก

Q1 ความซับซ้อนของโค้ดนี้คืออะไร? 
for (int i = 1; i < n; i *= 2)
    cout << i;
เฉลย

O(log n) — เพราะ i เพิ่มเป็น 1, 2, 4, 8, ... ค่าจะไปถึง n หลังจาก log₂(n) รอบ

Q2 โค้ดนี้? 
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j *= 2)  // ระวัง! j เริ่มที่ 0
        cout << i;
เฉลย

กับดัก! j *= 2 เมื่อ j = 0 จะได้ 0 ตลอด → loop in infinite ถ้าแก้เป็น j = 1 เริ่มต้น คำตอบคือ O(n log n)

Q3 ฟังก์ชัน recursive นี้? 
int f(int n) {
    if (n <= 1) return 1;
    return f(n/2) + f(n/2);
}
เฉลย

T(n) = 2·T(n/2) + O(1) ใช้ Master Theorem → O(n)

เพราะถึงจะแบ่งครึ่ง แต่เรียก 2 ครั้ง — รวมงานทั้งต้นไม้ = n

Q4 ใน labWeak2/findMax.cpp: 
int findMax(int arr[], int n) {
    if (n == 1) return arr[0];
    int maxRest = findMax(arr, n - 1);
    return (arr[n-1] > maxRest) ? arr[n-1] : maxRest;
}
เฉลย

เรียกตัวเอง 1 ครั้งโดยลด n ลง 1 + งานคงที่ → T(n) = T(n-1) + O(1) → O(n)
