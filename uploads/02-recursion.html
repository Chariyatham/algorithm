Recursion — ฟังก์ชันที่เรียกตัวเอง

Recursion = ฟังก์ชันแก้ปัญหาโดย เรียกตัวเองด้วยปัญหาที่เล็กลง ถ้าเข้าใจเรื่องนี้ลึก คุณจะเข้าใจ Divide & Conquer, DP, Backtracking, Graph DFS ทั้งหมดในชั่วพริบ

หลักคิด: 2 อย่างที่ต้องมีเสมอ

  1. Base case — เคสที่เล็กสุดที่ตอบได้ทันทีไม่ต้องเรียกตัวเอง ถ้าลืม → infinite recursion → stack overflow
  2. Recursive case — แก้ปัญหาขนาด n โดยเรียกตัวเองด้วย n ที่เล็กลง แล้วเอาผลลัพธ์มาประกอบ

Pattern พื้นฐาน

type solve(input) {
    if (base_case)
        return answer_directly;       // 1. base case

    // 2. recursive case: ลดปัญหาให้เล็กลง
    type sub_answer = solve(smaller_input);

    // 3. ประกอบคำตอบจาก sub_answer
    return combine(current_data, sub_answer);
}

ตัวอย่าง 1 — Factorial: n! = n · (n-1)!

int factorial(int n) {
    if (n <= 1) return 1;            // base case
    return n * factorial(n - 1);     // recursive case
}
// factorial(4) = 4 * factorial(3)
//              = 4 * 3 * factorial(2)
//              = 4 * 3 * 2 * factorial(1)
//              = 4 * 3 * 2 * 1 = 24

เห็นภาพ: Call Stack ทำงานยังไง?

ทุกครั้งที่เรียกฟังก์ชัน คอมจะ push frame ใหม่ ลง stack เมื่อ return จะ pop frame ออก กด Play เพื่อดู

กด Play เพื่อเริ่ม

ตัวอย่าง 2 — Fibonacci: f(n) = f(n-1) + f(n-2)

int fib(int n) {
    if (n <= 1) return n;            // base: fib(0)=0, fib(1)=1
    return fib(n - 1) + fib(n - 2);  // เรียก 2 ครั้ง!
}

เพราะเรียกตัวเอง 2 ครั้ง ต้นไม้ recursion จึงโตเป็น 2ⁿ — ดูได้ชัดในรูปด้านล่าง มี node สีแดง ที่คำนวณซ้ำ! (เราจะแก้ในหน้า DP)

ตัวอย่าง 3 — findMax (จาก labWeak2/findMax.cpp)

int findMax(int arr[], int n) {
    if (n == 1) return arr[0];                       // base: array 1 ตัว
    int maxRest = findMax(arr, n - 1);               // หา max ของ n-1 ตัวแรก
    return (arr[n-1] > maxRest) ? arr[n-1] : maxRest;
}

// เทียบ "ตัวสุดท้าย" กับ "max ของส่วนที่เหลือ"
// ลด n ทีละ 1 → T(n) = T(n-1) + O(1) → O(n)
Trace ตามมือ: input arr = [3, 7, 2, 9], n=4
  • findMax([3,7,2,9], 4) → เทียบ 9 กับ findMax([3,7,2], 3)
  • findMax([3,7,2], 3) → เทียบ 2 กับ findMax([3,7], 2)
  • findMax([3,7], 2) → เทียบ 7 กับ findMax([3], 1)
  • findMax([3], 1) → return 3 (base case)
  • return ขึ้นมา: max(7,3)=7, max(2,7)=7, max(9,7)=9 ✓

กฎเหล็ก 4 ข้อของ Recursion

  1. มี base case ทุกครั้ง — เช็คก่อนเรียกตัวเอง ไม่งั้น stack overflow
  2. การเรียก recursive ต้อง ลดขนาดปัญหา เข้าใกล้ base case
    เช่น solve(n-1), solve(n/2), solve(left, mid) ✓
    solve(n) หรือ solve(n+1) ✗ — วนไม่จบ
  3. เชื่อ recursion — assume solve(n-1) ทำงานถูก แล้วใช้ผลของมัน อย่าพยายาม trace ทุกชั้นในหัว
  4. ระวัง stack memory — แต่ละ call กิน stack ~MB n > 10,000 ในบาง compiler อาจ overflow

เปรียบเทียบ: Recursion vs Loop

แบบ recursionแบบ loop
int sumLoop(int a[], int n) {
    int s = 0;
    for (int i = 0; i < n; i++)
        s += a[i];
    return s;
}
 
int sumRec(int a[], int n) {
    if (n <= 0) return 0;
    return a[n-1] + sumRec(a, n-1);
}

ทั้งคู่ O(n) เหมือนกัน — ผลลัพธ์เท่ากัน แต่แบบ recursion กิน memory เพิ่ม เพราะใช้ call stack

แบบฝึก

Q1 เขียน power(b, e) ที่คำนวณ be แบบ recursive
เฉลย 
int power(int b, int e) {
    if (e == 0) return 1;          // base: anything^0 = 1
    return b * power(b, e - 1);    // b^e = b * b^(e-1)
}

Complexity: O(e) — ลด e ทีละ 1

Q2 ปรับ power ให้เป็น O(log e)
เฉลย 
int powerFast(int b, int e) {
    if (e == 0) return 1;
    int half = powerFast(b, e / 2);
    if (e % 2 == 0) return half * half;   // b^e = (b^(e/2))^2
    else return half * half * b;          // เลขคี่: คูณ b เพิ่มอีก 1 ตัว
}

เพราะแบ่งครึ่งทุกรอบ → O(log e) เร็วกว่ามาก!

เคล็ด: เก็บ half ไว้ตัวแปรเดียว ไม่ใช่เรียก recursive 2 ครั้ง — ไม่งั้นกลับเป็น O(e)

Q3 เขียน reverseStr(s) ที่กลับ string แบบ recursive
เฉลย 
string reverseStr(string s) {
    if (s.size() <= 1) return s;
    return reverseStr(s.substr(1)) + s[0];
    //     ^กลับส่วนที่เหลือ           ^เอาตัวแรกไปท้าย
}
// reverseStr("abc") = reverseStr("bc") + 'a'
//                   = (reverseStr("c") + 'b') + 'a'
//                   = ("c" + 'b') + 'a' = "cba"
Q4 ฟังก์ชันต่อไปนี้ถูกหรือไม่? ถ้าผิด แก้ยังไง? 
int countDigits(int n) {
    return countDigits(n / 10) + 1;
}
เฉลย

ผิด — ขาด base case! เมื่อ n = 0 จะเรียก countDigits(0) ตลอดไป → stack overflow

int countDigits(int n) {
    if (n < 10) return 1;          // เลขหลักเดียว = 1 หลัก
    return countDigits(n / 10) + 1;
}

(หรือถ้าต้องการให้ countDigits(0) = 0 เปลี่ยน base เป็น if (n == 0) return 0;)
