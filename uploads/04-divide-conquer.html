Divide & Conquer — แบ่ง · แก้ · รวม

"ปัญหาใหญ่แก้ไม่ได้ ก็แบ่งให้เล็กลงครึ่งนึง แก้ครึ่งนึงก่อน อีกครึ่งค่อยว่ากัน" หลักง่าย ๆ นี้ก่อให้เกิดอัลกอริทึมที่เร็วที่สุดในวงการ — Merge sort, Quick sort, Binary search, Strassen matrix multiply, FFT

Pattern 3 ขั้น

  1. Divide — แบ่งปัญหาเป็น sub-problem ที่เล็กลง (ปกติ ½)
  2. Conquer — แก้ sub-problem แต่ละอันแบบ recursive จนถึง base case (เล็กพอจะแก้ตรง ๆ)
  3. Combine — รวมคำตอบของ sub-problem เป็นคำตอบของปัญหาเดิม
type solve(problem) {
    if (problem เล็กพอ) return คำตอบตรง ๆ;     // base
    sub1, sub2 = divide(problem);                // 1. divide
    ans1 = solve(sub1);                          // 2a. conquer
    ans2 = solve(sub2);                          // 2b. conquer
    return combine(ans1, ans2);                  // 3. combine
}

ตัวอย่างที่ 1 — Binary Search

ค้นหา target ใน array ที่เรียงแล้ว

int binarySearch(vector<int>& a, int lo, int hi, int target) {
    if (lo > hi) return -1;                  // base: ไม่เจอ
    int mid = lo + (hi - lo) / 2;            // divide

    if (a[mid] == target) return mid;        // เจอ
    if (a[mid] < target)
        return binarySearch(a, mid+1, hi, target);  // หาฝั่งขวา
    else
        return binarySearch(a, lo, mid-1, target);  // หาฝั่งซ้าย
}

Recurrence: T(n) = T(n/2) + O(1) → O(log n)

เลือก target แล้วกด Play
lo hi mid เลิกพิจารณา เจอ!

ตัวอย่างที่ 2 — Merge Sort (recursion tree)

ดู หน้า Sorting สำหรับโค้ดเต็ม — ที่นี่เน้น เห็น tree

รอเริ่ม

Master Theorem — สูตรคิด complexity D&C

ถ้า recurrence รูปแบบ:

T(n) = a · T(n/b) + O(nd)

โดย a = จำนวน sub-problem, b = อัตราการแบ่ง, nd = งานนอก recursion (combine)

เงื่อนไขผลลัพธ์ตัวอย่าง
d < logb(a) O(nlogb(a)) Strassen: T(n) = 7T(n/2) + O(n²) → O(n2.81)
d = logb(a) O(nd log n) Merge sort: T(n) = 2T(n/2) + O(n) → O(n log n)
d > logb(a) O(nd) T(n) = 2T(n/2) + O(n²) → O(n²)
ข้อจำ: log2(2) = 1, log2(4) = 2, log2(7) ≈ 2.81, log2(8) = 3

วิเคราะห์ทีละกรณี

Merge Sort: T(n) = 2T(n/2) + O(n)

a=2, b=2, d=1 → log2(2)=1=d → กรณีที่ 2 → O(n log n)

Binary Search: T(n) = T(n/2) + O(1)

a=1, b=2, d=0 → log2(1)=0=d → กรณีที่ 2 → O(n0 log n) = O(log n)

Naive matrix multiply (recursive): T(n) = 8T(n/2) + O(n²)

a=8, b=2, d=2 → log2(8)=3 > d=2 → กรณีที่ 1 → O(n³) (เท่า iterative)

Strassen matrix multiply: T(n) = 7T(n/2) + O(n²)

a=7, b=2, d=2 → log2(7)≈2.81 > 2 → O(n2.81) เร็วกว่า naive!

กระจาย matrix n×n เป็น 4 บล็อก n/2 × n/2:

A = | A11  A12 |     B = | B11  B12 |
    | A21  A22 |         | B21  B22 |

C = A·B = | A11·B11 + A12·B21    A11·B12 + A12·B22 |

ตัวอย่างที่ 4 — Maximum Subarray (Kadane vs D&C)

หา subarray ที่ผลรวมสูงสุด เช่น [-2, 1, -3, 4, -1, 2, 1, -5, 4] → [4,-1,2,1] sum=6

int maxCrossingSum(vector<int>& a, int l, int m, int r) {
    int leftSum = INT_MIN, sum = 0;
    for (int i = m; i >= l; i--) { sum += a[i]; leftSum = max(leftSum, sum); }
    int rightSum = INT_MIN; sum = 0;
    for (int i = m+1; i <= r; i++) { sum += a[i]; rightSum = max(rightSum, sum); }
    return leftSum + rightSum;
}

int maxSubarray(vector<int>& a, int l, int r) {
    if (l == r) return a[l];                     // base
    int m = (l + r) / 2;
    int left = maxSubarray(a, l, m);             // 1. ซ้าย
    int right = maxSubarray(a, m+1, r);          // 2. ขวา
    int cross = maxCrossingSum(a, l, m, r);      // 3. ข้ามกลาง
    return max({left, right, cross});            // combine
}
// T(n) = 2T(n/2) + O(n) → O(n log n)
หมายเหตุ: ปัญหานี้ Kadane's algorithm แก้ได้ O(n) เร็วกว่า — แต่เป็นตัวอย่างที่เห็นภาพ D&C ชัดเจน

เทคนิคจำง่าย: เมื่อไหร่ใช้ D&C?

  • ปัญหา แบ่งครึ่งได้ตามธรรมชาติ (array แบ่ง ½, tree แบ่ง subtree ซ้าย-ขวา)
  • คำตอบของ sub-problem ประกอบเป็นคำตอบของปัญหาเดิมได้
  • sub-problem แต่ละอัน เป็นอิสระ (ไม่งั้นต้องใช้ DP — ดูบทถัดไป)

Q2 เขียน recursive function หา ค่าที่ปรากฏมากกว่า n/2 ครั้ง (majority element) แบบ D&C
เฉลย 
int countInRange(vector<int>& a, int x, int l, int r) {
    int c = 0;
    for (int i = l; i <= r; i++) if (a[i] == x) c++;
    return c;
}

int majority(vector<int>& a, int l, int r) {
    if (l == r) return a[l];                    // base
    int m = (l + r) / 2;
    int leftMaj = majority(a, l, m);
    int rightMaj = majority(a, m+1, r);
    if (leftMaj == rightMaj) return leftMaj;
    int lc = countInRange(a, leftMaj, l, r);
    int rc = countInRange(a, rightMaj, l, r);
    return (lc > rc) ? leftMaj : rightMaj;
}
// T(n) = 2T(n/2) + O(n) → O(n log n)
// (Boyer-Moore voting ทำได้ O(n))
Q3 ทำไม binary search ต้องการ array ที่เรียงแล้ว?
เฉลย

เพราะการตัดสินใจไปฝั่งซ้ายหรือขวาขึ้นอยู่กับ a[mid] < target ถ้า array ไม่เรียง ค่าฝั่งซ้าย/ขวาของ mid อาจมีอะไรก็ได้ — เลิกหวังเดาทิศได้

ค่าที่ คาดเดาได้ ของ array เรียงคือ "ทุกตัวฝั่งซ้าย mid ≤ a[mid]"

Q4 วิเคราะห์ Strassen วิธีนี้ทำไมเร็วกว่า naive
เฉลย

Naive: 8 multiply, 4 add → T(n) = 8T(n/2) + O(n²)

Strassen: ใช้สูตร 7 ตัว (M1..M7) แทน 8 multiply

โดยเสียค่า addition เพิ่มขึ้น (จาก 4 เป็น ~18) — แต่ multiply ครอบงำ ใน recursion

Master: 7T(n/2) + O(n²): a=7, b=2 → log2(7)≈2.81 → O(n2.81)

n=1000: naive ทำ 10⁹ ops, Strassen ~10⁸·⁴ — ช้ากว่าน้อยกว่า ~3 เท่า แต่ const factor สูง ในทางปฏิบัติคุ้มเฉพาะ n > ~100
